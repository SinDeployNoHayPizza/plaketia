# Open Industry Standards

## Commitment to Open Standards

Plaketia uses and produces open industry standards wherever possible. This ensures interoperability with existing EDA tools, manufacturers, and educational resources.

## Standards Supported

### SPICE Netlist Format

| Aspect | Detail |
|--------|--------|
| **Standard** | SPICE 3f5 / ngspice compatible netlist |
| **Use** | Primary simulation input/output format |
| **Status** | ✅ Fully supported from Phase 1 |

Plaketia reads and writes standard SPICE netlists. Any SPICE-compatible simulator (ngspice, LTspice, PSpice) can run the exported netlist.

Example:
```spice
* Generated by Plaketia
R1 N001 N002 10k
Q1 N003 N002 GND N002 2N2222
V1 N001 GND DC 12
.model 2N2222 NPN(Is=14.34f ...)
.op
.end
```

### Gerber RS-274X

| Aspect | Detail |
|--------|--------|
| **Standard** | Gerber RS-274X (Extended Gerber) |
| **Use** | PCB fabrication output |
| **Status** | 🔜 Planned for Phase 4 |

Export PCB layers as individual Gerber files:
- `.GTL` — Top copper
- `.GBL` — Bottom copper (for double-sided future)
- `.GTS` — Top solder mask
- `.GTO` — Top silkscreen
- `.GD1` — Drill data (Excellon format)

### IPC-2581

| Aspect | Detail |
|--------|--------|
| **Standard** | IPC-2581 (open, XML-based) |
| **Use** | Unified design-to-manufacturing data |
| **Status** | 🔮 Future (post-MVP) |

Single XML file containing layer stackup, netlist, component placement, and drill data.

### IPC-7351

| Aspect | Detail |
|--------|--------|
| **Standard** | IPC-7351 (component footprints) |
| **Use** | Footprint pad dimensions and land patterns |
| **Status** | ✅ Followed from Phase 1 (reference) |

Footprint dimensions follow IPC-7351 density levels (Most, Nominal, Least).

### SPICE Model Libraries

| Aspect | Detail |
|--------|--------|
| **Standard** | ngspice model syntax (.model statements) |
| **Use** | Device models for simulation |
| **Status** | ✅ Supported |

Models use standard ngspice/SPICE3 syntax:
- BJT: `Gummel-Poon` model
- MOSFET: `Shichman-Hodges` (Level 1), `BSIM3/4` (Level 8)
- Diode: Standard junction model
- OpAmp: Subcircuit-based models

### File Formats

| Format | Extension | Use | Phase |
|--------|-----------|-----|-------|
| JSON | `.plaketia` | Native project file | Phase 1 |
| SPICE netlist | `.cir`, `.net` | Simulation export/import | Phase 2 |
| Gerber RS-274X | `.gbr`, `.GTL`, etc. | PCB fabrication | Phase 4 |
| Excellon | `.drl` | Drill data | Phase 4 |
| SVG | `.svg` | Schematic export | Phase 2 |
| CSV | `.csv` | Bill of Materials | Phase 4 |
| PNG | `.png` | Screenshot (schematic, 3D) | Phase 3 |
| IPC-2581 | `.xml` | Unified PCB output | Future |

## Why Open Standards

1. **Vendor independence**: Users are not locked into a proprietary format
2. **Education**: Students can export and simulate in any SPICE tool
3. **Manufacturing**: Gerber files are accepted by all PCB manufacturers
4. **Interoperability**: Import from/export to KiCad, LTspice, etc.
5. **Longevity**: Open formats remain readable indefinitely
